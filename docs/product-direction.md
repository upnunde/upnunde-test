# 제품 방향 및 도메인 개요

## 제품 비전

이 서비스는 **창작자(크리에이터)가 시리즈를 만들고 에피소드를 지속적으로 쌓아가는 창작 운영 플랫폼**을 지향합니다.  
핵심은 “전문 도구(잉크 에디터)의 표현력”과 “일반 창작자도 바로 쓸 수 있는 쉬운 편집 경험”을 함께 제공하는 것입니다.

## 목적

크리에이터가 **인터랙티브 소설(노벨게임) 형식의 에피소드**를 쉽게 제작할 수 있도록, 시리즈·에피소드·리소스·**선형 블록 기반 스크립트**를 한 흐름에서 다루는 제작 도구를 제공합니다.

## 핵심 사용자 가치

- **쉬운 에디팅:** 기존 잉크 에디터의 난이도를 낮춘 편집 화면으로, 비전문 사용자도 에피소드를 작성·수정할 수 있게 한다.
- **명확한 구조:** 장면, 대사, 리소스, 선택지, 이벤트 구간을 블록 단위로 분리해 읽기·편집이 쉽게 한다.
- **안전한 편집:** 실행 취소/다시 실행, 임시저장 **스냅샷 히스토리**로 실수·되돌리기 비용을 낮춘다.
- **일관된 용어:** 블록 타입·라벨·검증 메시지를 한국어 기준으로 통일한다 (상세는 [ux-and-content-standards.md](./ux-and-content-standards.md)).

## 리소스 정책

- **창작자 등록 리소스 우선:** 에피소드 내용 구성의 중심은 사용자가 등록한 리소스(예: 캐릭터, 배경)다.
- **플랫폼 제공 리소스 병행:** BGM·효과음 등은 플랫폼 기본 제공 리소스를 함께 활용할 수 있다.
- **혼합 사용 전제:** 사용자 리소스 + 플랫폼 리소스가 자연스럽게 결합되도록 에디터/선택 UI를 설계한다.

## 도메인 개요

| 영역 | 설명 |
|------|------|
| **시리즈** | 작품 단위. 페르소나 등 시리즈 전역 메타와 연결된다. |
| **에피소드** | 회차 단위. 생성 폼(메타 입력)과 스크립트 에디터 뷰로 나뉜다. |
| **스크립트 블록** | `scene`, `text`, `choice`, `background`, `character` 등 타입별 한 줄(또는 메타데이터) 단위. 중첩 속성 없이 **평면 리스트**로 모델링한다. |
| **리소스** | 배경, 캐릭터, BGM, 갤러리 등 — 에디터 블록과 연동되는 자산 카탈로그. |
| **미리보기** | 모바일 프레임 등으로 연출·텍스트 흐름을 확인한다. |

## 화면 흐름 (에디터 앱 기준)

- **폼 뷰 (`form`):** 에피소드 메타 입력 등 생성 단계.
- **에디터 뷰 (`editor`):** 블록 편집, 씬 내비게이션, 미리보기.

상태는 클라이언트 스토어(`currentView` 등)로 전환되며, URL 쿼리(`view=form`)로 폼 진입을 유도할 수 있다.

## 단계별 목표 (로드맵)

### 1) 현재 단계: 제작 UX 정착

- 에피소드/캐릭터 등 등록 폼과 잉크 기반 에디터를 안정화한다.
- 임시저장, 히스토리, 검증, 미리보기 등 **핵심 작성 루프**를 완성한다.

### 2) 중기 단계: 크리에이터 스튜디오화

- 유튜브 스튜디오처럼 창작자가 한곳에서 운영할 수 있는 기능을 확장한다.
- 주요 대상: **통계, 정산, 에피소드 관리, 분석, 광고 관리**.

### 3) 장기 단계: AI 기반 자동 생성

- 현재의 수기 입력 필드를 점진적으로 AI 보조/자동화한다.
- 최종적으로는 창작자가 **핵심 키워드·요약 내용**만 입력해도 초안(에피소드/캐릭터/구성)을 생성할 수 있게 한다.
- AI 결과물은 자동 생성 후에도 사람이 수정 가능한 편집 흐름을 기본 원칙으로 유지한다.

## 비목표 (현 단계)

- 실제 **서버 영속 임시저장·배포 파이프라인**은 TODO로 남겨 두고, 클라이언트 히스토리·스낵바 등 **UX 골격**을 우선한다.
- MCP/Figma는 **에디터 본연의 스토리텔링 기능과 독립**으로 유지한다.
- 통계/정산/광고/AI 자동 생성은 방향성에 포함되지만, **현재 단계의 완료 기준**에는 포함하지 않는다.
