# 에디터·히스토리·임시저장 정책

스크립트 에디터와 관련된 **동작 기준**을 정리한다. 구현은 `app/src/store/useEditorStore.ts`, `app/src/components/editor/EditorSubHeader.tsx` 등을 참고한다.

## 블록 모델

- 블록은 **서로 독립**이며 **중첩 속성 없이** 리스트로 관리한다 (`ScriptBlock`).
- 타입별 의미는 `types/editor.ts`, 한글 라벨은 `blockTypeLabels.ts` / `blockLabelColors.ts`와 일치시킨다.
- **선택지(`choice`)** 는 `data.choices` 배열로 보관한다.
- **텍스트(`text`)** 는 `data.speaker`로 화자를 둔다. 기본값은 **「나레이션」**.

## 실행 취소 / 다시 실행

- `undoStack` / `redoStack`에 스냅샷을 쌓으며, 스택 깊이 상한(`MAX_UNDO`)으로 메모리를 제한한다.
- 블록 추가·수정·삭제·재정렬·타입 변경 등 편집 작업은 일관되게 undo 가능 경로를 탄다.

## 임시저장 스크립트 히스토리

- **트리거:** 사용자가 **임시저장**을 실행할 때 `addScriptHistoryEntry`가 호출되어, **현재 `blocks` 전체 스냅샷**이 히스토리에 쌓인다.
- **저장 단위:** 각 항목은 `id`, `blocks` 복제본, `savedAt`(타임스탬프)을 가진다.
- **개수 상한:** `MAX_SCRIPT_HISTORY`로 오래된 항목은 잘린다 (구현상 앞쪽이 최신 항목을 유지하는 방식).

### 목록 표시 순서 (정책)

- **가장 최근 임시저장이 목록 최상단**에 온다.
- UI에서는 `savedAt` 기준 **내림차순** 정렬을 사용해, 저장소 배열 순서와 무관하게 동일한 경험을 보장한다.
- 빈 상태 **데모 타임스탬프**도 동일하게 **최신이 위**로 정렬한다.
- 히스토리 시간 라벨은 [ux-and-content-standards.md](./ux-and-content-standards.md)의 **Global Policy(날짜·시간 표기)** 를 따른다.

### 히스토리 UI (팝오버)

- **너비:** 콘텐츠 영역은 `min(100vw - 여백, 240px)` 수준으로 제한해 좁은 패널에서도 과도하게 넓지 않게 한다.
- **헤더 영역:** 「히스토리」 라벨 위쪽 패딩은 시각적 균형을 위해 충분히 둔다.
- 각 행에서 **불러오기**로 해당 스냅샷을 에디터에 적용하고, 팝오버는 닫힌다.

## 검증 (등록 등)

- 빈 콘텐츠가 허용되지 않는 블록 타입, 선택지 필수 필드, `event` / `event_end` 개수 일치 등 **제출 전 검증** 규칙이 있다.
- 검증 실패 시 등록 버튼 등이 비활성화되는 등 **차단 + 명확한 피드백**을 우선한다.

## 내비게이션·나가기

- 저장되지 않은 변경이 있으면 **뒤로 가기** 시 확인 다이얼로그를 띄울 수 있다.
- **임시저장 없이 나가기**와 **임시저장 후 이동**의 선택지를 제공하는 흐름을 유지한다.

## 향후 연동 (방향)

- 실제 API 연동 시에도 **히스토리의 의미(시점별 스냅샷, 최신 우선 표시)** 는 동일하게 유지하는 것을 권장한다.
