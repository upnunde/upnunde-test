# AI 역할군 + n8n 자동화 운영 가이드

이 문서는 이 저장소에서 AI 역할군을 분리하고, 역할 간 핸드오프를 n8n으로 자동화하기 위한 기준입니다.

## 1) 현재 기본 역할 (P0)

| 역할 | 핵심 책임 | 입력 | 산출물 |
|---|---|---|---|
| 기획 | 요구사항, 예외 케이스, 수용 기준(AC) 확정 | 제품 목표, 제약사항 | PRD, AC, 예외 목록 |
| UIUX | 화면 구조/상태/카피 일관성 정리 | PRD, 디자인 시스템 | 화면 명세, 상태 정의(빈/로딩/오류) |
| FE | 구현/리팩터링/버그 수정 | PRD, UI 명세 | 코드, 구현 노트, 검증 결과 |

## 2) 확장 역할 우선순위

| 우선순위 | 역할 | 추가 트리거 | 추가 후 필수 산출물 |
|---|---|---|---|
| P1 | QA | 회귀 버그 반복, 배포 직전 안정화 필요 | 테스트 케이스, 결함 리포트 |
| P1 | PM | 병렬 과제 증가, 우선순위 충돌 | 우선순위 보드, 의사결정 로그 |
| P2 | BE | 목데이터 한계, API/DB 필요 | API 스펙, 구현/마이그레이션 |
| P3 | DevOps | 배포 실패 반복, 운영 모니터링 필요 | 배포/롤백 절차, 모니터링 기준 |
| P3 | 보안 | 인증/개인정보/결제 도입 | 보안 체크리스트, 위험 평가 |

## 3) 역할 핸드오프 기본 규칙

1. 기획 -> UIUX -> FE 순서로 진행한다.
2. 각 단계 시작 전 입력 문서 링크를 명시한다.
3. 각 단계 종료 시 "완료 조건 3개"를 체크한다.
4. 단계 전환은 n8n 알림(슬랙/디스코드/이메일)로 남긴다.

## 4) n8n 적용 범위 (초기)

- 역할 전환 알림: 기획 완료 -> UIUX 시작, UIUX 완료 -> FE 시작
- PR 오픈 알림: FE PR 생성 시 QA 체크리스트 자동 전파
- 일일 상태 요약: 당일 작업 항목/막힘 이슈 자동 요약

## 5) 로컬 실행

1. `cp n8n/.env.example n8n/.env`
2. `npm run n8n:up`
3. 브라우저 `http://localhost:5678` 접속
4. `n8n/workflows/role-handoff-checklist.json` 를 Import 해서 템플릿으로 사용

## 6) 권장 운영 방식

- 같은 채팅에서도 가능하지만, 역할 혼선을 줄이려면 대화를 분리한다.
  - 기획 채팅
  - UIUX 채팅
  - FE 채팅
- n8n은 "역할 자체"를 정하는 도구가 아니라, 역할 간 전달을 자동화하는 도구로 사용한다.
